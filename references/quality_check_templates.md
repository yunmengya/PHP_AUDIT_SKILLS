# Quality Check Verification Templates (Mandatory Fill-In)

> This file is the sole verification source for the quality-checker Agent. The QC inspector MUST fill in the "Actual" and "Status" columns line by line; summarizing or omitting any check item in a single sentence is MUST NOT.

---

## ⛔ 6 Iron Rules (violating any one → automatic FAIL verdict)

| # | Iron Rule | Description |
|---|-----------|-------------|
| R1 | **MUST NOT add sections** | The verification report MUST only contain sections defined in this template; adding any sections or subsections is MUST NOT |
| R2 | **MUST NOT delete/reorder sections** | All sections MUST appear in the order defined by this template; skipping, merging, or swapping is MUST NOT |
| R3 | **MUST NOT modify table columns** | The verification table columns (#/Check Item/Expected/Actual/Status) MUST NOT be added, removed, or renamed |
| R4 | **MUST NOT leave placeholders** | All 【填写】 MUST be replaced with actual content before submission; `grep '【填写】'` hit count MUST be 0 |
| R5 | **MUST NOT merge multiple reports** | Each verification MUST produce an independent report; mixing results from multiple agents under verification in the same report is MUST NOT |
| R6 | **MUST NOT modify check item titles** | The text in the "Check Item" column is fixed; the QC inspector MAY only fill in the "Actual" and "Status" columns |

**Automated detection script (the QC inspector MUST execute before submission):**
```bash
# Detect leftover placeholders
grep -rn '【填写】\|TODO\|TBD\|PLACEHOLDER' "$QC_REPORT" && echo "❌ R4 违规：存在残留占位符" && exit 1
# Detect JSON syntax (if applicable)
python3 -m json.tool "$OUTPUT_FILE" > /dev/null 2>&1 || echo "❌ JSON 语法错误"
# Detect encoding
file --mime-encoding "$QC_REPORT" | grep -qE 'utf-8|us-ascii' || echo "❌ 编码非 UTF-8"
```

## Standard Report Structure

Every verification report MUST contain the following three parts — none MAY be omitted:

````markdown
# Verification Report: {Agent Under Verification Name}

## Basic Information
- QC Inspector: quality-checker-{N}
- Verification Target: {Agent Under Verification Name}
- Phase: {Phase-1 / Phase-2 / Phase-3 / Phase-4 / Phase-4.5 / Phase-5}
- Verified Files: {actual output file paths read, comma-separated if multiple}
- Associated Schema: {corresponding schemas/*.schema.json file path, N/A if none}

## Item-by-Item Verification Results

{Use the corresponding phase checklist table below, fill in "Actual" and "Status" columns line by line}

## Final Verdict
- Status: **✅ PASS** / **❌ FAIL**
- Passed Items: {M}/{N}
- Failed Items List (if any):
  - {#}. {Check Item Name}: {specific missing content and remediation requirements}
- Remediation Requirements: {for failed items, what the agent under verification MUST supplement}
````

> **Iron Rule**: Every report generated by the QC inspector MUST strictly follow this structure. Missing any part → report is invalid and MUST be regenerated.

## Cross-Phase Data Consistency Verification

> This section is executed by the final QC inspector (quality-checker-final) after Phase 5 verification. It validates data flow consistency and completeness across multiple phases, ensuring no hallucination and no omissions.

### Rule 1: P0 Routes ↔ Exploit Full Coverage

**Verification Logic:** Every Sink with `priority=P0` in `priority_queue.json` MUST have a corresponding result in `exploits/`.

| # | Check Item | Expected | Actual | Status |
|:----:|--------|------|------|:----:|
| C1 | P0 Sink full coverage | P0 count in priority_queue = P0 result count in exploits | 【填写：P0=?个, 已覆盖=?个】 | ⬜ |
| C2 | P1 Sink full coverage | P1 count in priority_queue = P1 result count in exploits | 【填写：P1=?个, 已覆盖=?个】 | ⬜ |

| C4 | No omission declaration | Untested P0 MUST have skip reason (not_applicable + reason) | 【填写】 | ⬜ |

**Verification Script:**
```bash
P0_TOTAL=$(jq '[.[] | select(.priority=="P0")] | length' "$WORK_DIR/priority_queue.json")

```
